# Han Unification and Unihan

## Han Unification

- Define a single code point for characters that are considered the same across CJK languages
- Characters that are considered the same but traditionally visually rendered slightly differently across CJK languages are considered allographs: different glyphs representing the same grapheme (only the grapheme is encoded and gets a code point)
  - Note: characters that are rendered differently in itraditional Chinese and simplified Chinese are not unified, that is, they get separate code points
- Specific variations of a character/code point (such as traditional Chinese, Japanese, etc.) can be explicitly indicated by appending a variation selector character to a character
  - This allows mixing different variants of characters in the same text



> Notes:
>  - Grapheme vs. sememe distinction mentioned in the Wikipedia article is ignored for simplicity
